General Information

Each of the following 5 pages will present you with 2 screenshots:

The first screenshot is from the Jupyter Notebook [2]. By clicking on the [2], you can see it live on the GitHub, from which it is readily available since the publication of SWARM-SLR [1] in 2024.

Screenshots look like this:

Some recurring elements are:

  • The title of the current step
  • Some requirements to be met in achieving it
  • A description what needs to be done
  • Potentially a code block to present template text such as LLM prompts 
  • An exemplary LLM reply to be expected when entering the example prompt 

 

You will then be shown the screenshot of the SWARM-SLR AIssistant implementation of the same step. These screenshots are taken from the AIssistant, an application currently under development and not yet otherwise publically available. These screenshots typically look as follows:

Some recurring elements are:

  • The title of the current step in the step panel on the left and on top of the central interface
  • The step description below the step title on top of the central interface
  • The input field on the bottom of the central interface
  • The LLM Chat in the center of the central interface
  • The output assets on the right panel, which are generated in each step, stored and reused in later steps.

Based on these screenshots, you are then asked to compare the Jupyter Notebook guidance to the AIssistant implementation, using 8 metrics, e.g. "complicated or easy", or "not interesting or interesting". 

A free-form text field allows you to add any further feedback you may want to provide.

Future Work: Open Tool Registry

The survey is almost done - if you are interested in future work, here are 2 final questions, otherwise, click next down below, and thank you for your contribution!

 

For the future, we seek to reach another major milestones:

An AIssistant Tool Registry, similar to bio.tools [3], which replaces the previous SWARM-SLR tool assessment [4]. It will be open for anyone to contribute tools, and SWARM-SLR will just be one of many different workflows supported by it. By merging these two frameworks into one, we hope to provide better tool support and reduce redundant work. Linked to an extension browser similar to Visual Studio Code (VSC), installing another tool should be much simpler and truly realize the vision of a modular, customized workflow in the AIssistant.
